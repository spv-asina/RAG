## 1. Запуск системы

### Быстрый запуск (Docker)

```bash
docker run -d --name rag-backend -p 5000:5000 \
  -v $(pwd)/data:/app/data:ro \
  -e LLM_ENABLED=false \
  --memory=2g \
  rag-backend:latest
```

После запуска откройте браузер и перейдите по адресу:

➡️ **http://localhost:5000**

### Если Docker не установлен

```bash
pip install -r backend/requirements.txt
python serve.py
# → http://localhost:5000
```

### Как проверить, что система работает

Выполните в терминале:

```bash
curl http://localhost:5000/health
```

Ожидаемый ответ:
```json
{"status": "ok", "chunks": 42, "sessions": 0, "llm_available": false}
```

## 2. Чат-режим: вопросы по учебнику

Система умеет отвечать на вопросы по теории графов, находя ответы в учебнике (`data/chapter.md`).

### Как задать вопрос

1. Откройте http://localhost:5000 в браузере
2. Перейдите на вкладку **"Чат"** (если интерфейс разделён на вкладки)
3. Введите вопрос в текстовое поле и нажмите Enter

### Примеры вопросов

| Вопрос                                              | Что вернёт система                                         |
|-----------------------------------------------------|------------------------------------------------------------|
| «Что такое граф?»                                   | Определение графа как совокупности V и E                   |
| «Объясни понятие эйлерова цикла»                    | Описание цикла, проходящего по каждому ребру 1 раз         |
| «Что такое матрица смежности?»                      | Определение матрицы смежности с примером                   |
| «Чем отличается цепь от простой цепи?»              | Объяснение различий (повторение вершин)                   |
| «Дай определение изолированной вершины»             | Вершина степени 0                                          |
| «Расскажи про поиск в глубину»                      | Алгоритм обхода графа                                      |

### Как это работает «под капотом»

1. Система определяет ключевой термин в вашем вопросе
2. Ищет определение этого термина в специальном индексе (построен при старте)
3. Если термин не найден — выполняет TF-IDF поиск по всем чанкам учебника
4. Если уверенность низкая (<30%) — расширяет запрос связанными терминами
5. При наличии LLM — улучшает формулировку ответа

### Что означают индикаторы ответа

- **Высокая уверенность** (зелёный) — score ≥ 0.3, система уверена в ответе
- **Средняя уверенность** (жёлтый) — score ≥ 0.15, ответ найден, но возможны неточности
- **Низкая уверенность** (красный) — score < 0.15, ответ может быть нерелевантен

## 3. Режим тестирования

### Запуск теста

1. Перейдите на вкладку **"Тренажёр"** (Trainer)
2. Нажмите **«Сгенерировать тест»** (Generate)
3. Система создаст тест из **5 вопросов** по случайным терминам

### Типы вопросов

#### 📝 Definition (Открытый вопрос)

**Формат:** «Что такое {термин}?» или «Дайте определению понятию ...»

**Как отвечать:**
- Напишите своими словами определение термина
- Чем больше ключевых терминов совпадёт с учебником — тем выше шанс

**Критерий оценки:**
- TF-IDF сходство с эталонным определением
- Пересечение ключевых слов
- Композитный порог: `0.6 × similarity + 0.4 × overlap ≥ 0.12`

---

#### ⚖️ True/False (Верно/Неверно)

**Формат:** «Верно ли, что [утверждение]?»

**Как отвечать:** выберите **«Верно»** или **«Неверно»**

**Особенности:**
- Утверждение может быть истинным (из учебника) или ложным (с заменой термина)
- При неверном ответе показывается правильное утверждение

---

#### 🔘 Multiple Choice (Множественный выбор)

**Формат:** Показывается определение, нужно выбрать соответствующий термин из 4 вариантов.

**Пример:**
```
Какой термин соответствует определению:
«Граф, у которого степени всех вершин равны...»?

A) Полный граф
B) Регулярный граф      ← правильный ответ
C) Двудольный граф
D) Связный граф
```

---

#### ✏️ Fill Blank (Заполнить пропуск)

**Формат:** Определение с пропуском (`__________`)

**Варианты:**
- **Полный термин:** заменяется весь термин (например, «Мостом называется __________...»)
- **Первое слово:** для составных терминов (например, «...называется __________ подграфа...»)
- **Последнее слово:** (например, «...называется стягивание __________»)

**Совет:** Введите слово или термин **в той же грамматической форме**, что и в определении.

---

#### 🔗 Matching (Соответствие)

**Формат:** Определение с замаскированным термином (все вхождения заменены на `_____` с учётом падежа)

**Пример:**
```
Какой термин соответствует следующему определению?
«_____ называется совокупность двух множеств — непустого
множества V и множества E...»
```

Правильный ответ: Граф

---

### Проверка ответов

После каждого ответа система показывает:
- ✅ или ❌ — верно/неверно
- **Прогресс** — сколько отвечено из 5
- **Правильный ответ** (при ошибке)
- **Детали оценки** для открытых вопросов (similarity %, overlap %, порог)

### После завершения теста

Когда все 5 вопросов отвечены, система покажет:
- **Итоговый балл**: X из 5
- **Потраченное время**
- **Детализацию** по каждому вопросу: вопрос, ваш ответ, правильный ответ

## 4. Как проверяются ответы

### Открытые вопросы (definition)

Ответ сравнивается с эталонным определением из учебника:

```
1. TF-IDF векторизация ответа и эталона
2. Косинусное сходство (similarity_score) — 60% веса
3. Пересечение множеств слов (overlap_score) — 40% веса
4. Итог: final_score ≈ 0.12 → ✔️
```

**Порог срабатывания** специально низкий (0.12), чтобы засчитывать ответы своими словами.

### Закрытые вопросы (true_false, multiple_choice, fill_blank, matching)

- Точное сравнение со значением термина (регистр не важен)
- Для `true_false`: сравнивается с `truth_value` («Верно» / «Неверно»)

## 5. Отслеживание прогресса

Система запоминает ваши результаты в рамках одной сессии (вкладки браузера).

### Что отслеживается

Для каждого из 57 терминов:
- Сколько раз спрашивали (`asked_count`)
- Сколько раз ответили верно (`correct_count`)
- Какие типы вопросов использовались

### Статусы терминов

| Статус         | Иконка | Значение                                        |
|----------------|--------|-------------------------------------------------|
| `never_asked`  | 🔵     | Термин ещё не встречался в тестах               |
| `always_wrong` | 🔴     | Спрашивали, но ни одного правильного ответа     |
| `in_progress`  | 🟡     | Есть и правильные, и неправильные ответы        |
| `learned`      | 🟢     | Хотя бы один правильный ответ (изучен)          |

### Где смотреть прогресс

- На вкладке «Прогресс» (Progress) интерфейса
- Через API: `GET /api/trainer/progress` (с заголовком `X-Session-Id`)

### Как «выучить» термин

Достаточно **один раз правильно ответить** на вопрос по термину — он перейдёт в статус `learned`. Умный выбор терминов будет реже предлагать изученные термины и чаще — те, где вы ошибаетесь.

## 6. Советы и рекомендации

### Как получить лучший ответ в чате

- **Формулируйте вопрос чётко**, с указанием термина: «Что такое X?» вместо «Расскажи про это»
- Вопросы на русском языке обрабатываются лучше (морфология настроена на русский)
- Если ответ не найден — перефразируйте вопрос, используйте синонимы

### Как лучше проходить тесты

1. **Начните с чата** — прочитайте определения интересующих терминов
2. **Сгенерируйте тест** — проверьте понимание
3. **Анализируйте ошибки** — система показывает правильный ответ
4. **Повторите** — новый тест будет включать «слабые» термины
5. **Следите за прогрессом** — цель: все 57 терминов в статусе `learned`

### Если LLM отключён

- Вопросы формулируются по шаблонам: «Что такое {термин}?»
- Ответы — точные цитаты из учебника (без переформулирования)
- Система остаётся полностью работоспособной

### Если LLM включён

- Вопросы и ответы становятся более «человечными»
- LLM обучен не менять смысл — только улучшать формулировки
- Если LLM недоступен (не запущен контейнер) — система автоматически переходит в режим без LLM

## 7. Нестандартные ситуации

### Система не находит ответ на вопрос

Возможные причины:
- Термин отсутствует в `terms.txt` (список 57 терминов)
- Определение в `chapter.md` не содержит маркеров («называется», «— это»)
- Вопрос сформулирован слишком обще (например, «расскажи про графы»)

**Решение:** переформулируйте вопрос, укажите конкретный термин.

### Тест закончился досрочно (закрыли вкладку)

- Сессия активна 1 час после последнего запроса
- При возврате на страницу в той же вкладке сессия восстановится (если не прошёл час)
- Текущий тест сохраняется — можно продолжить

### Хочу начать тест заново

- Нажмите **«Сбросить тест»** (Reset) в интерфейсе
- Или отправьте: `POST /api/trainer/reset`

### Не помню свой session_id

- При открытии страницы фронтенд генерирует новый UUID
- Старая сессия будет потеряна (если не скопировать session_id вручную)
- В здоровом режиме это не требуется — просто продолжайте работу в той же вкладке
