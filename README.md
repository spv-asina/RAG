# RAG по теории графов

**Retrieval-Augmented Generation** система по дискретной математике (теория графов) с вопросно-ответным чатом и генератором тестов.

Система ищет ответы в учебнике через TF-IDF, а при наличии локальной LLM (Qwen 2.5 7B) улучшает формулировки вопросов и ответов. Без LLM работает полностью автономно.

---

## Возможности

- **Чат по учебнику**: задайте вопрос по теории графов — система найдёт релевантный фрагмент и вернёт ответ
- **Генерация тестов**: автоматически создаёт тест из 5 вопросов разных типов по 57 терминам
- **Проверка ответов**: TF-IDF валидация для открытых вопросов, точное совпадение для закрытых
- **Сессии**: одна вкладка = одна сессия, история сохраняется in-memory (1 час)
- **LLM-улучшение** (опционально): подключите Qwen 2.5 7B для более связных формулировок
- **Graceful degradation**: без LLM система полностью работоспособна на шаблонах и TF-IDF

---

## Технологии

| Компонент | Технология |
|-----------|-----------|
| **Backend** | Python 3.11+, Flask, scikit-learn (TF-IDF, cosine similarity) |
| **Морфология** | pymorphy3 (лемматизация русского языка) |
| **Frontend** | Vanilla HTML + CSS + JS (single-page application) |
| **LLM** | Qwen 2.5 7B Q4_K_M через llama.cpp (опционально) |
| **Инфраструктура** | Docker, docker-compose, gunicorn |

## Быстрый старт

### Запуск без Docker

```bash
# Установка зависимостей
pip install -r backend/requirements.txt

# Запуск dev-сервера
python serve.py

# Открыть в браузере
# http://localhost:5000
```

### Запуск с Docker (только бэкенд, без LLM)

```bash
LLM_ENABLED=false docker-compose up backend
```

### Запуск с Docker + LLM

1. **Скачайте модель** Qwen 2.5 7B Q4_K_M (~4.5 ГБ):

```bash
mkdir -p models
wget -O models/qwen2.5-7b-q4_k_m.gguf \
  https://huggingface.co/Qwen/Qwen2.5-7B-GGUF/resolve/main/qwen2.5-7b-q4_k_m.gguf
```

2. **Запустите оба сервиса**:

```bash
LLM_ENABLED=true docker-compose up --build
```

LLM-сервер стартует дольше (загрузка модели в память). После появления `http://localhost:5000` в браузере — система готова.

### Использование сессий

Сессия идентифицируется заголовком `X-Session-Id`. Фронтенд генерирует UUID при загрузке страницы и передаёт его в каждом запросе.

```bash
# Пример запроса к чату
curl -X POST http://localhost:5000/api/chat \
  -H 'Content-Type: application/json' \
  -H 'X-Session-Id: a1b2c3d4-e5f6-7890-abcd-ef1234567890' \
  -d '{"message": "Что такое эйлеров цикл?"}'
```

## Типы вопросов в тесте

| Тип | Описание | Валидация |
|-----|----------|-----------|
| `definition` | «Что такое {term}?» — открытый ответ | TF-IDF косинусное сходство + пересечение ключевых слов (порог ≥0.12) |
| `true_false` | Верно/Неверно — утверждение на основе определения | Точное совпадение |
| `multiple_choice` | 1 правильный из 4 вариантов | Точное совпадение |
| `fill_blank` | Заполнить пропуск в определении | Точное совпадение (регистронезависимое) |
| `matching` | Какой термин соответствует определению? | Точное совпадение (регистронезависимое) |

## Архитектура

### Как работает поиск (retriever.py)

1. **Сегментация**: учебник разбивается на структурированные чанки с заголовками разделов
2. **Лемматизация**: все слова приводятся к нормальной форме через pymorphy3 (однократно при старте)
3. **TF-IDF**: строится матрица 15000 n-грамм (1-2), субlinear TF
4. **Бустинг**: заголовки разделов ×1.5, совпадение терминов ×1.3
5. **Определения терминов**: отдельный индекс с точным поиском по маркерам («называется», «— это» и др.)
6. **Fallback**: при низкой уверенности запрос расширяется смежными терминами

### Сессии

- Одна вкладка браузера = одна сессия (session_id генерируется на фронтенде)
- Состояние хранится in-memory на сервере
- TTL = 1 час бездействия, после чего сессия удаляется
- В сессии хранится: история чата, состояние текущего теста

### LLM-улучшение (опциональное)

- Подключается через HTTP к llama.cpp серверу (OpenAI-compatible API)
- **Улучшает формулировки** вопросов и ответов (без изменения смысла)
- **Генерирует дистракторы** для тестов
- **Безопасность**: модель не принимает решений о правильности, валидатор проверяет сохранность ключевых терминов
- При недоступности LLM система продолжает работать на шаблонах и TF-IDF

## Разработка

### Добавление нового термина

1. Добавьте термин в `data/terms.txt` (одна строка — один термин)
2. Найдите или напишите определение в `data/chapter.md`
3. Перезапустите сервер — retriever перестроит индекс терминов

### Запуск тестов

В проекте нет отдельного набора тестов — проверка осуществляется через ручное тестирование API или через интерфейс в браузере.

### Локальный запуск без Docker (режим разработки)

```bash
# Установка
pip install -r backend/requirements.txt

# Запуск с авто-перезагрузкой при изменениях
FLASK_DEBUG=1 python serve.py
```
